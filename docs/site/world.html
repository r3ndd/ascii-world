World Module

World, chunk, and map management. Handles terrain data, entity storage per chunk, and distance-based update scheduling for large open worlds.

World

The main world container that manages chunks and coordinates all world-related systems.

new World()
constructor(width: number, height: number, chunkSize: number, ecsWorld: ECSWorld) 
const world = new World(1000, 1000, 64, ecsWorld);
world.initialize();
getTileAt()
getTileAt(x: number, y: number): Tile | null 
const tile = world.getTileAt(100, 200);
if (tile && tile.blocksMovement) {
    console.log('Cannot move here!');
}
setTileAt()
setTileAt(x: number, y: number, tile: Tile): boolean 
world.setTileAt(100, 200, TERRAIN.wall);
isValidPosition()
isValidPosition(x: number, y: number): boolean 
if (world.isValidPosition(newX, newY)) {
    // Can move here
}
setPlayerPosition()
setPlayerPosition(x: number, y: number): void
Updates player position and activates/deactivates chunks accordingly. 
world.setPlayerPosition(playerX, playerY);
getEntitiesAt()
getEntitiesAt(x: number, y: number): Entity[] 
const entities = world.getEntitiesAt(100, 200);
for (const entity of entities) {
    console.log(`Entity ${entity.id} is here`);
}

Chunk

64x64 tile container with local coordinate system.

getTile()
getTile(localX: number, localY: number): Tile | null 
const tile = chunk.getTile(10, 15);
setTile()
setTile(localX: number, localY: number, tile: Tile): boolean 
chunk.setTile(10, 15, TERRAIN.floor);
toWorldPosition()
toWorldPosition(localX: number, localY: number): Position 
const worldPos = chunk.toWorldPosition(10, 15);
// Returns { x: chunkX * 64 + 10, y: chunkY * 64 + 15 }
toLocalPosition()
toLocalPosition(worldX: number, worldY: number): Position

ChunkManager

Manages chunk lifecycle, activation, and distance-based updates.

getOrCreateChunk()

worldToChunk()
worldToChunk(worldX: number, worldY: number): ChunkCoord

Terrain

Built-in terrain types with default properties.

Terrain Char Blocks Movement Blocks Light Description
TERRAIN.floor . No No Default floor
TERRAIN.wall # Yes Yes Solid wall
TERRAIN.water ~ No No Water
TERRAIN.tree T Yes No Tree (transparent)
TERRAIN.door + Yes Yes Door
TERRAIN.stairs > No No Stairs

UpdateScheduler

Distance-based update scheduling for performance optimization.

Update Frequency by Distance

Distance Update Interval Description
0-1 chunks Every turn Full updates
2-3 chunks Every 5 turns Course updates
4-6 chunks Every 20 turns Course updates
7+ chunks Every 100 turns Course updates

Complete Example

import { World, TERRAIN } from 'ascii-world';

// Create world
const world = new World(1000, 1000, 64, ecsWorld);
world.initialize();

// Get tile
const tile = world.getTileAt(100, 100);
console.log(tile?.char);  // '.' for floor

// Set custom tile
world.setTileAt(100, 100, TERRAIN.wall);

// Check position validity
if (world.isValidPosition(500, 500)) {
    // Can move here
}

// Update player position (activates nearby chunks)
world.setPlayerPosition(playerX, playerY);

// Get entities at position
const entities = world.getEntitiesAt(100, 200);
for (const entity of entities) {
    console.log(`Found entity: ${entity.id}`);
}
Performance Note: Chunks are lazily created when accessed. Only chunks near the player are kept active (3x3 grid by default), minimizing memory usage in large worlds.
