ECS Module

Entity Component System (ECS) architecture for flexible game object composition. Entities are containers for components, and systems process entities with specific component combinations.

ECSWorld

The central coordinator that manages entities and systems. Provides entity lifecycle management and efficient queries.

new ECSWorld()
constructor(eventBus?: EventBus) 
import { ECSWorld } from 'ascii-world';

const ecsWorld = new ECSWorld();
// Or with custom event bus:
const ecsWorld = new ECSWorld(customEventBus);
createEntity()
createEntity(): Entity
Creates a new entity with a unique auto-incrementing ID. 
const player = ecsWorld.createEntity();
removeEntity()
removeEntity(entityId: EntityId): boolean
Removes an entity and notifies all systems. Returns true if successful. 
ecsWorld.removeEntity(player.id);
queryEntities()
queryEntities(query: Query): Entity[]
Queries entities based on component filters. Results are cached for performance.

Query Options

  • all - Entity must have ALL these components
  • any - Entity must have at least ONE of these components
  • none - Entity must have NONE of these components 
// Find all entities with position and health
const combatants = ecsWorld.queryEntities({
    all: ['position', 'health']
});

// Find all visible renderable entities
const visible = ecsWorld.queryEntities({
    all: ['position', 'renderable'],
    none: ['invisible']
});
addSystem()
addSystem(system: System): void
Adds a system to the world. Systems are sorted by priority (lower = earlier). 
ecsWorld.addSystem(new MovementSystem());
ecsWorld.addSystem(new CombatSystem());

Entity

A game object container. Entities have no behavior themselves - all functionality comes from components.

addComponent()
addComponent<T extends Component>(component: T): Entity
Adds a component to the entity. Returns the entity for chaining. Overwrites existing components of the same type. 
entity
    .addComponent(createPosition(10, 20))
    .addComponent(createHealth(100, 100))
    .addComponent(createRenderable('@', '#ffffff'));
getComponent()
getComponent<T>(type: string): T | undefined 
const position = entity.getComponent<PositionComponent>('position');
if (position) {
    console.log(`Entity is at (${position.x}, ${position.y})`);
}
hasComponent()
hasComponent(type: string): boolean 
if (entity.hasComponent('health')) {
    // Entity has health component
}
removeComponent()
removeComponent(type: string): boolean 
entity.removeComponent('buff');

System

Systems process entities with specific component combinations. Implement the System interface or extend BaseSystem. 

import { System, Entity } from 'ascii-world';

class MovementSystem implements System {
    readonly name = 'MovementSystem';
    readonly priority = 10;

    query = {
        all: ['position', 'velocity']
    };

    update(entities: Entity[], deltaTime: number): void {
        for (const entity of entities) {
            const pos = entity.getComponent<PositionComponent>('position');
            const vel = entity.getComponent<VelocityComponent>('velocity');

            if (pos && vel) {
                pos.x += vel.vx * deltaTime;
                pos.y += vel.vy * deltaTime;
            }
        }
    }

    onEntityAdded?(entity: Entity): void {
        console.log(`Entity ${entity.id} added to MovementSystem`);
    }

    onEntityRemoved?(entity: Entity): void {
        console.log(`Entity ${entity.id} removed from MovementSystem`);
    }
}

BaseSystem

Convenience abstract class for creating systems:

import { BaseSystem, Entity } from 'ascii-world';

class HealthSystem extends BaseSystem {
    readonly name = 'HealthSystem';
    readonly priority = 5;
    query = { all: ['health'] };

    update(entities: Entity[], deltaTime: number): void {
        for (const entity of entities) {
            const health = entity.getComponent<HealthComponent>('health');
            if (health && health.current <= 0) {
                // Handle death
            }
        }
    }
}

Built-in Components

Position Component

interface PositionComponent extends Component {
    type: 'position';
    x: number;
    y: number;
    z: number;  // Layer/elevation (0 = ground level)
}

// Factory function
const pos = createPosition(10, 20);       // z defaults to 0
const upperPos = createPosition(10, 20, 1); // Layer 1 (upper floor)
entity.addComponent(pos);

Health Component

interface HealthComponent extends Component {
    type: 'health';
    current: number;
    max: number;
}

const health = createHealth(100, 100);
entity.addComponent(health);

Speed Component

interface SpeedComponent extends Component {
    type: 'speed';
    value: number;  // 100 = normal speed
}

const speed = createSpeed(150);  // 50% faster than normal

Actor Component

interface ActorComponent extends Component {
    type: 'actor';
    isPlayer: boolean;
    energy: number;
    nextAction?: string;
}

const actor = createActor(true);  // true = is player

Renderable Component

interface RenderableComponent extends Component {
    type: 'renderable';
    char: string;
    fg: string;
    bg?: string;
}

const renderable = createRenderable('@', '#ffffff', '#000000');

Component Factories

Function Signature Description
createPosition (x, y, z?) Position in world coordinates (z = layer)
createVelocity (vx, vy) Velocity vector
createHealth (current, max) Health points
createSpeed (value) Speed value (100 = normal)
createActor (isPlayer?) Actor for turn system
createRenderable (char, fg, bg?) Visual representation
createInventoryComponent (capacity, volume) Inventory for items

Events

Event Data Description
ecs:entityCreated { entityId } New entity created
ecs:entityRemoved { entityId } Entity removed
Performance Tip: Entity queries are cached. The cache is invalidated when entities are added, removed, or when components change. For best performance, minimize entity churn in hot paths.
