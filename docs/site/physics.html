Physics Module

Collision detection, movement, field of view (FOV), lighting, and pathfinding. Uses rot.js for FOV, lighting, and pathfinding algorithms.

PhysicsSystem

Handles entity movement and collision detection with the world.

new PhysicsSystem()
constructor(world: World, ecsWorld: ECSWorld, eventBus: EventBus) 
const physics = new PhysicsSystem(gameWorld, ecsWorld, eventBus);
canMoveTo()
canMoveTo(x: number, y: number): boolean 
if (physics.canMoveTo(newX, newY)) {
    // Safe to move
}
moveEntity()
moveEntity(entity: Entity, direction: Direction): boolean
Moves entity in a direction (8-way). Returns true if successful. 
// Move in cardinal directions
physics.moveEntity(player, 'north');
physics.moveEntity(player, 'south');
physics.moveEntity(player, 'east');
physics.moveEntity(player, 'west');

// Move diagonally
physics.moveEntity(player, 'northeast');
physics.moveEntity(player, 'southwest');
moveEntityTo()
moveEntityTo(entity: Entity, x: number, y: number): boolean 
physics.moveEntityTo(player, 100, 200);

for (const pos of visibleTiles) {
    console.log(`Can see tile at (${pos.x}, ${pos.y})`);
}
isVisible()
isVisible(x: number, y: number): boolean 
if (fov.isVisible(enemyX, enemyY)) {
    // Enemy is visible to player
}
isExplored()
isExplored(x: number, y: number): boolean 
// Check if tile was ever visible
if (fov.isExplored(x, y)) {
    // Show terrain (but not entities) in "fog of war"
}

FOV with Rendering

// In your render loop
fov.computeFOV(playerX, playerY, 10);

// Only render visible tiles
for (let y = 0; y < mapHeight; y++) {
    for (let x = 0; x < mapWidth; x++) {
        if (fov.isVisible(x, y)) {
            // Render tile normally
            renderTile(x, y);
        } else if (fov.isExplored(x, y)) {
            // Render in "fog" (darker)
            renderTileFog(x, y);
        }
        // Otherwise don't render (unexplored area)
    }
}

Pathfinding

A* and Dijkstra pathfinding using rot.js algorithms.

new Pathfinding()
constructor(world: World) 
const pathfinding = new Pathfinding(gameWorld);
findPath()
findPath(startX: number, startY: number, endX: number, endY: number): Position[] | null
A* pathfinding to a specific destination. 
const path = pathfinding.findPath(100, 100, 150, 150);

if (path) {
    for (const pos of path) {
        console.log(`Move to (${pos.x}, ${pos.y})`);
    }
} else {
    console.log('No path found!');
}
findPathDijkstra()
findPathDijkstra(startX: number, startY: number, targetCallback: (x: number, y: number) => boolean): Position[] | null
Dijkstra pathfinding to find a target matching a condition. 
// Find path to nearest water tile
const path = pathfinding.findPathDijkstra(
    playerX,
    playerY,
    (x, y) => {
        const tile = gameWorld.getTileAt(x, y);
        return tile?.terrain === 'water';
    }
);

if (path) {
    // Follow path to water
}

Complete Example

import {
    PhysicsSystem,
    FOVSystem,
    LightingSystem,
    Pathfinding
} from 'ascii-world';

// Initialize systems
const physics = new PhysicsSystem(world, ecsWorld, eventBus);
const fov = new FOVSystem(world);
const lighting = new LightingSystem(world);
const pathfinding = new Pathfinding(world);

// Player movement with arrow keys
document.addEventListener('keydown', (e) => {
    switch(e.key) {
        case 'ArrowUp':
            physics.moveEntity(player, 'north');
            break;
        case 'ArrowDown':
            physics.moveEntity(player, 'south');
            break;
        case 'ArrowLeft':
            physics.moveEntity(player, 'west');
            break;
        case 'ArrowRight':
            physics.moveEntity(player, 'east');
            break;
    }

    // Update FOV after movement
    const pos = player.getComponent<PositionComponent>('position');
    fov.computeFOV(pos.x, pos.y, 10);

    // Update lighting
    lighting.moveLightSource('player_light', pos.x, pos.y);
});

// AI pathfinding to player
const aiPath = pathfinding.findPath(
    enemyPos.x, enemyPos.y,
    playerPos.x, playerPos.y
);

if (aiPath && aiPath.length > 1) {
    const nextStep = aiPath[1];
    // Move enemy towards player
}
Note: All FOV, lighting, and pathfinding use rot.js implementations. This provides robust, optimized algorithms for roguelike games.
