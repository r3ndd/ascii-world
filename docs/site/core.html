Core Module

The Core module provides the foundation of the ASCII World Engine, including the main Engine class, configuration management, event system, and common type definitions.

Engine

The main entry point for the game engine. Manages initialization, lifecycle, and coordinates between all systems.

new Engine()
constructor()
Creates a new Engine instance. Uses the global event bus and loads configuration automatically. 
const engine = new Engine();
initialize()
async initialize(): Promise<void>
Initializes the engine, loading configuration and emitting initialization events. 
await engine.initialize();
start()
start(): void
Starts the engine and emits the 'engine:started' event. 
engine.start();
stop()
stop(): void
Stops the engine and emits the 'engine:stopped' event. 
engine.stop();

EventBus

Pub/sub event system for decoupled communication between systems. The engine provides a global event bus instance.

on()
on<T>(event: string, handler: (data: T) => void): () => void
Registers an event handler. Returns an unsubscribe function.

Parameters

  • event - Event name to listen for
  • handler - Function called when event is emitted

Returns

Function to unsubscribe from the event

const unsubscribe = eventBus.on('entity:moved', (data) => {
    console.log(`Entity ${data.entityId} moved to (${data.x}, ${data.y})`);
});

// Later, to unsubscribe:
unsubscribe();
emit()
emit<T>(event: string, data: T): void
Emits an event to all registered handlers. 
eventBus.emit('player:joined', { playerId: 1, name: 'Hero' });

Global Event Bus

A singleton event bus is available for application-wide events: 

import { globalEventBus } from 'ascii-world';

globalEventBus.on('game:started', () => {
    console.log('Game started!');
});

Config

Singleton configuration manager for world settings, action costs, and debug flags.

getInstance()
static getInstance(): Config
Returns the singleton Config instance. 
const config = Config.getInstance();
console.log(config.world.width);  // 1000

Configuration Properties

Property Type Description
world.width number World width in tiles (default: 1000)
world.height number World height in tiles (default: 1000)
world.chunkSize number Chunk size in tiles (default: 64)
world.maxActiveChunks number Max chunks to update (default: 9)
debug boolean Enable debug logging

interface Bounds {

Common Events

Event Data Description
engine:initialized { version: string } Engine finished initialization
engine:started {} Engine started
engine:stopped {} Engine stopped
Tip: Use the EventBus for all cross-system communication. This keeps your code decoupled and testable.
