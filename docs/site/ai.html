AI Module

The AI system provides intelligent behavior for NPCs using behavior trees, memory management, and sensory systems. Designed with extensibility in mind, it supports future integration with GOAP (Goal-Oriented Action Planning).

AISystem

The main ECS system that coordinates AI processing. Manages behavior trees, memory systems, and integrates with the physics and pathfinding systems.

new AISystem()
constructor(ecsWorld, physicsSystem, pathfinding, eventBus) 
import { AISystem } from 'ascii-world';

const aiSystem = new AISystem(
    ecsWorld,
    physicsSystem,
    pathfinding,
    eventBus
);

ecsWorld.addSystem(aiSystem);

Behavior Trees

Behavior trees provide a hierarchical structure for AI decision-making. The system is designed for GOAP compatibility, allowing future integration with goal-oriented planning.

Node Status

enum NodeStatus {
    SUCCESS = 'success',   // Node completed successfully
    FAILURE = 'failure',   // Node failed
    RUNNING = 'running'    // Node is still executing
}

Blackboard

Shared memory storage for behavior tree nodes. Provides type-safe key-value storage and world state checking for GOAP compatibility.

Blackboard
new Blackboard(entity: Entity) 
const blackboard = new Blackboard(entity);
blackboard.set('targetEntity', player);
blackboard.set('inCombat', true);

const target = blackboard.get<Entity>('targetEntity');
const isCombat = blackboard.getOrDefault('inCombat', false);

Composite Nodes

Sequence

Executes children in order until one fails. Succeeds only if all children succeed.

const patrolSequence = new Sequence()
    .addChild(new HasTarget())
    .addChild(new MoveToTarget(pathfinding))
    .addChild(new AttackTarget());

Selector

Executes children in order until one succeeds. Fails only if all children fail.

const combatSelector = new Selector()
    .addChild(new CanAttackTarget())
    .addChild(new MoveToTarget(pathfinding));

Parallel

Executes all children simultaneously with configurable success/failure policies.

const parallel = new Parallel(
    ParallelPolicy.REQUIRE_ONE_SUCCESS,
    ParallelPolicy.REQUIRE_ONE_FAILURE
)
    .addChild(new ScanForEntities(8))
    .addChild(new UpdateTargetFromMemory(RelationshipType.HOSTILE));

Decorator Nodes

Decorator Description
Inverter Inverts child's result (SUCCESS ↔ FAILURE)
Succeeder Always returns SUCCESS
Failer Always returns FAILURE
Repeater Repeats child N times (or indefinitely)
UntilFail Repeats until child fails

Building a Behavior Tree

import { BehaviorTree, Sequence, Selector } from 'ascii-world';

function createHunterTree(): BehaviorTree {
    const tree = new Selector();
    
    // Combat branch
    const combat = new Sequence()
        .addChild(new CanSeeHostile())
        .addChild(new UpdateTargetFromMemory(RelationshipType.HOSTILE))
        .addChild(new Selector()
            .addChild(new CanAttackTarget())
            .addChild(new MoveToTarget(pathfinding))
        )
        .addChild(new AttackTarget());
    
    // Flee branch
    const flee = new Sequence()
        .addChild(new IsHealthLow(0.3))
        .addChild(new SetBlackboardValue(BBKeys.SHOULD_FLEE, true))
        .addChild(new MoveAway());
    
    // Wander branch
    const wander = new Repeater(-1)
        .setChild(new MoveRandom());
    
    tree
        .addChild(combat)
        .addChild(flee)
        .addChild(wander);
    
    return tree;
}

Memory System

Tracks memories for AI entities including entity memories, locations, events, and items. Memories decay over time based on importance and can be reinforced.

Memory Types

Type Description
MemoryType.ENTITY Another actor in the world
MemoryType.LOCATION A point of interest
MemoryType.EVENT Something that happened (combat, trade, etc.)
MemoryType.ITEM An item seen or obtained
MemoryType.TERRAIN A terrain feature

Relationship Types

enum RelationshipType {
    HOSTILE = 'hostile',
    FRIENDLY = 'friendly',
    NEUTRAL = 'neutral',
    AFRAID = 'afraid',
    CURIOUS = 'curious',
    ALLIED = 'allied'
}

Memory Importance

Level Value Description
TRIVIAL 1 Will forget quickly
LOW 2 Short-term memory
NORMAL 3 Standard memory duration
HIGH 5 Significant event, longer retention
CRITICAL 10 Life-threatening, never forget

Creating Custom Behaviors

Custom Action

import { Action, NodeStatus, BehaviorContext } from 'ascii-world';

class HealSelf extends Action {
    tick(context: BehaviorContext): NodeStatus {
        const health = context.entity.getComponent<HealthComponent>('health');
        
        if (!health || health.current >= health.max) {
            return NodeStatus.FAILURE;
        }
        
        // Restore health
        health.current = Math.min(health.max, health.current + 20);
        
        return NodeStatus.SUCCESS;
    }
    
    getCost(): number {
        return 2; // Higher cost for GOAP planning
    }
}

Custom Condition

import { Condition, BehaviorContext } from 'ascii-world';

class IsNightTime extends Condition {
    check(context: BehaviorContext): boolean {
        // Check world state through blackboard
        const worldTime = context.blackboard.get<number>('worldTime');
        return worldTime !== undefined && (worldTime < 600 || worldTime > 1800);
    }
}

Events

Event Data Description
ai:entityAdded { entityId } AI entity added to system
ai:entityRemoved { entityId } AI entity removed from system
ai:tick { entityId, status } Behavior tree ticked (for debugging)
GOAP Compatibility: The AI system is designed with future GOAP integration in mind. Behavior trees can be replaced with GOAP planners, and the blackboard system supports world state checking required for planning.
Performance Tip: Behavior trees are ticked every frame for active entities. Keep tree logic lightweight and use conditions to avoid expensive operations when not needed.
