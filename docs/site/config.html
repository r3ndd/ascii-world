Configuration

Global configuration management for world defaults, action costs, and debug settings. Use the Config singleton to access and modify engine settings.

Config

Singleton configuration manager providing access to all engine settings.

Config.getInstance()
static getInstance(): Config
Get the singleton Config instance. 
import { Config } from 'ascii-world';

const config = Config.getInstance();
console.log(config.world.width);  // 1000
config.world
get world(): typeof WORLD_DEFAULTS
Access world configuration defaults. 
const config = Config.getInstance();
console.log(`World size: ${config.world.width}x${config.world.height}`);
console.log(`Chunk size: ${config.world.chunkSize}`);
console.log(`Max active chunks: ${config.world.maxActiveChunks}`);
config.actions
get actions(): typeof ACTION_COSTS
Access action cost definitions. 
const config = Config.getInstance();
console.log(`Move cost: ${config.actions.MOVE}`);
console.log(`Attack cost: ${config.actions.ATTACK}`);
config.debug
get debug(): boolean
set debug(value: boolean)
Enable or disable debug mode for verbose logging. 
// Enable debug mode
Config.getInstance().debug = true;

// Check debug status
if (Config.getInstance().debug) {
    console.log('Debug mode enabled');
}

WORLD_DEFAULTS

Default world generation and update settings. Located in src/config/WorldDefaults.ts.

Property Value Description
width 1000 World width in tiles
height 1000 World height in tiles
chunkSize 64 Tiles per chunk (64x64)
chunkCount { x: 16, y: 16 } Computed: ceil(width/chunkSize)
maxActiveChunks 9 3x3 grid around player
fullUpdateDistance 1 Chunks updated every turn (distance 0-1)
courseUpdateSchedule Array Distance-based update intervals

CourseUpdateSchedule

Defines how often distant chunks are updated to save performance. 

interface CourseUpdateLevel {
    minDist: number;      // Minimum chunk distance
    maxDist: number;      // Maximum chunk distance
    interval: number;     // Turns between updates
}

// Default schedule:
[
    { minDist: 2, maxDist: 3, interval: 5 },      // Distance 2-3: every 5 turns
    { minDist: 4, maxDist: 6, interval: 20 },     // Distance 4-6: every 20 turns
    { minDist: 7, maxDist: Infinity, interval: 100 }  // Distance 7+: every 100 turns
]

ACTION_COSTS

Base action costs in ticks for the turn system. Located in src/config/ActionCosts.ts. Actual cost is divided by actor speed (speed 100 = normal).

Action Cost Description
MOVE 100 Movement to adjacent tile
ATTACK 100 Melee or ranged attack
CRAFT 300 Crafting an item
WAIT 50 Wait/pass turn
INTERACT 100 Interact with object/door
PICKUP 50 Pick up item from ground
DROP 25 Drop item on ground
USE 100 Use item (potion, scroll, etc.)
EQUIP 50 Equip gear
UNEQUIP 25 Unequip gear

Action Cost Calculation

Final action cost is adjusted by actor speed:

// Formula: cost = baseCost / (speed / 100)
const baseCost = ACTION_COSTS.MOVE;  // 100

// Speed 100 (normal): 100 / 1 = 100 ticks
// Speed 50 (slow): 100 / 0.5 = 200 ticks
// Speed 200 (fast): 100 / 2 = 50 ticks

function calculateCost(baseCost: number, speed: number): number {
    return Math.floor(baseCost / (speed / 100));
}

Modifying Configuration

To customize engine behavior, create your own configuration files or modify values at runtime:

Custom World Settings

// Create a smaller world for testing
import { Config } from 'ascii-world';

const config = Config.getInstance();

// Note: These should be set before creating the World
// as they are used during initialization
(config as any)._config.world = {
    ...config.world,
    width: 500,
    height: 500,
    chunkSize: 32,
    maxActiveChunks: 4  // 2x2 grid for smaller worlds
};

Custom Action Costs

// Make combat faster
const config = Config.getInstance();
(config as any)._config.actions = {
    ...config.actions,
    ATTACK: 75,   // Faster attacks
    MOVE: 100,    // Normal movement
    CRAFT: 200    // Faster crafting
};

GameConfig Interface

Complete configuration structure:

interface GameConfig {
    world: {
        width: number;
        height: number;
        chunkSize: number;
        chunkCount: { x: number; y: number };
        maxActiveChunks: number;
        fullUpdateDistance: number;
        courseUpdateSchedule: CourseUpdateLevel[];
    };
    actions: {
        [actionType: string]: number;
    };
    debug: boolean;
}

Best Practices

  • Set config early: Modify configuration before creating World or other systems
  • Use constants: Import WORLD_DEFAULTS and ACTION_COSTS as starting points
  • Test with debug: Enable config.debug = true during development
  • Balance costs: Action costs directly affect game pace and feel
  • Chunk sizing: Larger chunks use more memory but reduce chunk management overhead

Complete Example

import { Config, Engine, World, ECSWorld } from 'ascii-world';
import { WORLD_DEFAULTS } from 'ascii-world/config';

// Customize configuration before game initialization
const config = Config.getInstance();
config.debug = true;  // Enable debug logging

// Create engine with custom settings
const engine = new Engine();
await engine.initialize();

// Create ECS world
const ecsWorld = new ECSWorld();

// Create world (uses config values)
const world = new World(
    config.world.width,      // 1000
    config.world.height,     // 1000
    config.world.chunkSize,  // 64
    ecsWorld
);

// Check action costs
console.log(`Move cost: ${config.actions.MOVE} ticks`);
console.log(`Craft cost: ${config.actions.CRAFT} ticks`);

// Debug info
if (config.debug) {
    console.log('World configuration:', config.world);
    console.log('Active chunk grid:',
        Math.sqrt(config.world.maxActiveChunks), 'x',
        Math.sqrt(config.world.maxActiveChunks)
    );
}

Configuration File Locations

File Description

Performance Tip: For very large worlds (2000x2000+), consider increasing maxActiveChunks to maintain smooth performance around the player while keeping distant chunks dormant.
