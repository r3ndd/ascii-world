ASCII World Engine

ASCII World is a 2D ASCII game engine built on top of rot.js, designed for creating open-world survival games in the style of CDDA (Cataclysm: Dark Days Ahead).

Features

Entity Component System

Modern ECS architecture for flexible game object composition and efficient queries with caching.

Chunk-Based World

64x64 tile chunks with distance-based updates for optimal performance in large worlds.

Turn-Based Combat

Speed-based turn system using rot.js Scheduler.Speed for variable actor speeds.

Procedural Generation

Built-in generators for wilderness, dungeons, caves, and cities with pluggable architecture.

Mod Support

Load JSON content packs and JavaScript mods with dependency resolution.

Save/Load System

JSON-based persistence with compression support and save slot management.

Quick Example

import { Engine, ECSWorld, World, DisplayManager, Camera, Renderer } from 'ascii-world';

// Initialize the engine
const engine = new Engine();
await engine.initialize();

// Create ECS world
const ecsWorld = new ECSWorld();

// Setup display
const display = new DisplayManager({
    width: 80,
    height: 25,
    fontSize: 16
});

// Create game world
const gameWorld = new World(1000, 1000, 64, ecsWorld);
gameWorld.initialize();

// Create player entity
const player = ecsWorld.createEntity()
    .addComponent(createPosition(500, 500))
    .addComponent(createHealth(100, 100))
    .addComponent(createSpeed(100))
    .addComponent(createRenderable('@', '#ffffff'));

engine.start();

Architecture Overview

The engine is organized into distinct modules, each handling a specific aspect of game development:

  • Core - Engine initialization, configuration, and event bus
  • ECS - Entity Component System for game objects
  • Display - Rendering and viewport management via rot.js
  • World - Chunk management, terrain, and map systems
  • Time - Turn management and deferred updates
  • Physics - Movement, collision, FOV, lighting, pathfinding
  • Items - Item system with templates, inventory, and equipment
  • Content - Mod loading, world generation, and content packs

Default World Configuration

Parameter Value Description
World Size 1000x1000 tiles Total world dimensions
Chunk Size 64x64 tiles Tiles per chunk
Total Chunks 256 (16x16) Chunk grid for 1000x1000 world
Active Chunks 9 (3x3) Chunks updated each turn around player

Dependencies

  • rot.js ^2.2.0 - Rendering, FOV, scheduling, pathfinding
  • TypeScript ^5.0.0 - Type safety and modern JS features
Note: This engine is built specifically around the rot.js framework. All rendering, field of view, lighting, and pathfinding use rot.js implementations.
